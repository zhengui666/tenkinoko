# Security Policy

Tenkinoko 是一个与真实资金交易系统相关的代码仓库。这里的“安全”不仅指传统意义上的漏洞，也包括任何可能导致以下后果的问题：

- 风险约束被绕过
- 恢复路径失效
- 订单状态不一致
- 回测出现未来数据泄漏
- 凭证、账户信息或敏感配置被暴露

## Supported Scope

当前仓库仍处于活跃开发阶段。我们欢迎报告所有会影响下列方面的问题：

- 身份凭证或密钥处理
- 订单执行与状态机一致性
- RocksDB 恢复、重放与持久化安全
- 数据源完整性与失败路径
- 回测无泄漏保证
- 依赖安全问题

## Reporting A Vulnerability

请不要公开提交包含敏感细节的 issue。

如果你发现以下类型问题，请私下报告：

- API key / secret 泄露
- 可导致错误下单、重复下单、跳过风控的缺陷
- 可利用的恢复 / 重放不一致
- 可导致历史回测结论失真的严重泄漏问题

联系邮箱：

- `zhangzeyuan@sensetime.com`

## What To Include

请尽量包含以下信息：

- 受影响模块
- 触发条件
- 影响范围
- 最小复现步骤
- 是否涉及密钥、账户或真实资金风险
- 你建议的缓解方式

## Response Expectations

维护者会优先处理高风险问题，尤其是涉及：

- 实盘资金安全
- 风控绕过
- 状态恢复错误
- 数据泄漏

在问题未修复前，请避免公开披露可直接利用的细节。
